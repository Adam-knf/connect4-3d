## 需求概述

优化棋盘悬停交互体验，提升用户放置棋子的视觉反馈。

---

## 详细需求

### 1. 网格线简化
**现状问题**：网格线覆盖底部、顶层、垂直柱线，视觉杂乱

**需求**：
- 只保留底部网格线（Y=0 平面）
- 删除顶层网格线
- 删除垂直柱线

**验收标准**：
- 棋盘只显示底部一层网格线
- 视觉清晰，不遮挡棋子

---

### 2. 悬停检测逻辑调整
**现状问题**：悬停检测基于顶层格子或底座格子，无法正确处理棋子遮挡

**需求**：
- 从鼠标位置发射 Raycaster
- 计算鼠标在棋盘空间内的投影位置
- 根据投影位置确定对应的底部格子坐标 (x, z)
- 忽略棋子遮挡（穿透检测）

**验收标准**：
- 鼠标悬停在棋盘任意位置（包括棋子上方），都能正确识别对应的底部格子
- 悬停位置准确对应视觉上的格子

---

### 3. 底部格子高亮
**现状问题**：高亮格子位置和效果不够明显

**需求**：
- 高亮底部格子（Y=0 平面）
- 高亮颜色：使用配置中的强调色
- 高亮方式：发光效果（emissive）

**验收标准**：
- 悬停时底部格子明显发光
- 高亮颜色与视觉风格一致

---

### 4. 竖直空间网格线浮现
**新增功能**：

**需求**：
- 悬停格子对应的竖直空间（从底部到顶层）显示网格线
- 网格线淡入淡出效果
- 网格线发光效果（颜色、亮度可配置）
- 网格线半透明，不遮挡棋子

**配置项**（新增到 gameConfig.ts）：
```typescript
RENDER_CONFIG.verticalHighlight = {
  color: 0x3d9eff,       // 发光颜色（#3d9eff）
  opacity: 0.3,          // 透明度
  emissiveIntensity: 0.5, // 发光强度
}
```

**验收标准**：
- 悬停时竖直空间网格线浮现
- 离开时网格线消失
- 发光效果柔和，不刺眼

---

### 5. 虚影棋子 + 明显高亮
**现状问题**：预览棋子位置和高亮不够明显

**需求**：
- 虚影棋子显示在**实际将放置的位置**（顶层可用空位）
- 虚影棋子颜色根据当前玩家（黑/白）
- 虚影棋子半透明（opacity 0.4）
- 虚影棋子对应的底部格子高亮更明显（emissiveIntensity 更高）

**验收标准**：
- 悬停时虚影棋子清晰显示在正确位置
- 底部格子高亮比普通悬停更明显
- 离开时虚影棋子和高亮消失

---

## 技术要点

### 悬停检测实现思路
1. 从鼠标位置发射 Raycaster
2. 与棋盘空间平面（Y=boardHeight）做投影计算
3. 计算投影点的 (x, z) 坐标
4. 映射到底部格子坐标

### 竖直空间网格线实现思路
1. BoardRenderer 新增 `highlightVerticalColumn(x, z)` 方法
2. 创建 4 条垂直线（从 Y=0 到 Y=boardHeight）
3. 使用 LineBasicMaterial + emissive 效果
4. 悬停时添加到场景，离开时移除
