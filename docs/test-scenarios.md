# AI测试棋谱

**版本**: v1.0
**日期**: 2026-04-26
**用途**: AI评估算法分级验证

## 分级原则

- **EASY级别**: 单步决策，立即可见结果
- **MEDIUM级别**: 需要搜索2-3步，识别威胁模式
- **HARD级别**: 复杂局面，需要深度搜索和多因素权衡

**测试规则**: 高难度AI必须通过低难度测试

---

## EASY级别测试（depth=2）

### E-1: 立即获胜（必过）
**棋盘**: AI白棋3连待获胜
```
白: (0,0), (1,0), (2,0)
黑: 无
```
**测试**: AI应选择(3,0)获胜
**分数**: WIN=100000

### E-2: 阻挡对手获胜（必过）
**棋盘**: 黑棋3连待获胜
```
白: 无
黑: (0,0), (1,0), (2,0)
```
**测试**: AI应选择(3,0)阻挡
**分数**: BLOCK_WIN=5000

### E-3: 选择中心位置
**棋盘**: 空棋盘
**测试**: AI应倾向选择中心区域(2,2)
**分数**: 中心加分生效

### E-4: 立即获胜优先级（最高）
**棋盘**: AI有立即获胜位置，同时有其他高分位置
```
白: (0,0), (0,1), (0,2)  ← 可立即获胜(0,3)
     (1,1), (2,2)        ← 其他高分位置（两条2连）
黑: 少量棋子
```
**测试**: 立即获胜分数应高于任何其他位置
**分数**: WIN + maxDepth*1000 + 1000 > 任何威胁分数
**规则**: 所有难度必过

## MEDIUM级别测试（depth=3）

### M-1: 双3连威胁（必胜模式）
**棋盘**: AI白棋形成双威胁
```
白: (0,0), (1,0), (2,0)  ← 横向3连
     (0,0), (0,1), (0,2)  ← 纵向3连
黑: 少量阻挡
```
**测试**: AI应识别双威胁，任何一端都能获胜
**分数**: DOUBLE_THREAT_OWN=500

### M-2: 阻挡双威胁（紧急防守）
**棋盘**: 黑棋形成双威胁
```
白: 少量棋子
黑: (0,0), (1,0), (2,0)
     (0,0), (0,1), (0,2)
```
**测试**: AI应阻挡（无法完全阻挡时选择最有价值的位置）
**分数**: DOUBLE_THREAT_BLOCK=1000

### M-3: 两条2连=潜在双威胁
**棋盘**: AI白棋两条2连
```
白: (1,1), (2,2)  ← 对角2连
     (2,2), (3,2)  ← 横向2连
黑: 少量阻挡
```
**测试**: AI应识别两条2连的高价值位置
**分数**: POTENTIAL_DOUBLE_OWN=300

### M-4: 阻挡2连威胁
**棋盘**: 黑棋2连威胁
```
白: (1,1)
黑: (2,2), (2,1)  ← Y方向2连
```
**测试**: AI应阻挡(2,0)或(2,3)
**分数**: TWO_BLOCK=40，但需要Minimax识别威胁发展

### M-5: 叉子检测
**棋盘**: 中心位置形成叉子
```
白: (2,2)
黑: 少量棋子
```
**测试**: AI应识别(2,2)是多线交汇点，扩展潜力高
**分数**: FORK_BASE=150

## HARD级别测试（depth=4）

### H-1: 多步获胜路径
**棋盘**: 中盘复杂局面
```
白: 多颗棋子，无明显立即获胜
黑: 多颗棋子
```
**测试**: AI应找到4步内的获胜路径
**分数**: WIN + depth*1000

### H-2: 攻防抉择
**棋盘**: AI有发展潜力，对手也有威胁
```
白: (1,2), (2,2)  ← 有潜力
黑: (2,1), (2,0)  ← Y方向2连威胁
```
**测试**:
- 进攻位置(1,2)分数 vs 防守位置(2,3)分数
- AI需要权衡：阻挡分数 vs 自身发展分数
**分数**: 需要Minimax搜索评估后续发展

### H-3: 陷阱识别
**棋盘**: 对手设陷阱
```
白: 有诱惑性高分位置
黑: 布下陷阱，若AI贪高分则后续必输
```
**测试**: AI不应贪图表面高分，应识别陷阱
**分数**: Minimax深度搜索应揭示陷阱

## 日志案例分析

### Log-1: medium_1.log（立即获胜被忽略）
**问题描述**: AI明明可以立即获胜(0,4)，却选择了(1,2)
**根因**: WIN分数累加威胁加分后，非立即赢位置分数更高
**修复**: WIN分数按depth递减，立即赢最高分

### Log-2: medium_2.log（未阻挡2连）
**棋盘状态**:
```
Step 1: 玩家(2,2) 黑
Step 2: AI(1,1) 白
Step 3: 玩家(2,1) 黑 ← 形成(2,2)-(2,1) Y方向2连
Step 4: AI选择(1,2) 而不是阻挡位置
```
**评估分数**:
- (1,2) score=-9
- (2,3) score=-16
- (2,0) score=-27

**问题分析**:
- 玩家2连威胁分数(TWO_BLOCK=40)未在Minimax中产生足够影响
- AI认为(1,2)进攻位置分数更高(-9 vs -16)
- **属于M-4测试场景**

**潜在修复方向**:
- 提高2连阻挡的权重
- 或确保Minimax搜索中正确评估后续威胁发展
