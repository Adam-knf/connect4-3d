## 需求概述

开发一款基于Web的3D四子棋游戏，玩家在5x5x6三维空间棋盘上与AI对战，采用重力规则放置棋子，目标是任意三维方向连成4子获胜。包含2-3层难度AI、本地战绩记录、胜负特效与UI信息面板功能，视觉简约，支持自由视角切换。棋盘高度可扩展至8层。

*注：用户系统、排行榜功能暂缓，后续版本再做。*

---

## 游戏规则（已确认）

### 棋盘规格
- **长宽**：6×6 = 36列
- **高度**：默认6层，可扩展至8层
- **总容量**：216格（6层）或288格（8层）

### 棋子放置规则（重力规则）
- 玩家左键点击顶层空位，棋子自动落到该列最底层可用位置
- 棋子可在已有棋子上堆叠，不能悬浮放置
- 类似传统四子棋，扩展到三维空间

### 胜利条件
连成4子的所有有效方向：

| 连线类型 | 说明 | 方向向量示例 |
|----------|------|-------------|
| 水平横线 | 同层，X方向变化 | (+1, 0, 0) |
| 水平竖线 | 同层，Y方向变化 | (0, +1, 0) |
| 平面对角线 | 同层，XY对角 | (±1, ±1, 0) |
| 垂直线 | 跨层，仅Z变化 | (0, 0, +1) |
| 跨层斜线(XZ) | XZ平面对角 | (±1, 0, ±1) |
| 跨层斜线(YZ) | YZ平面对角 | (0, ±1, ±1) |
| 空间对角线 | 三维全变化 | (±1, ±1, ±1) |

总共13个方向向量，可通过三维数组统一检测。

### 平局条件
- 棋盘全满且无连线时判定平局
- 无时间限制

## 目标用户

- **主要用户**：休闲游戏玩家、策略游戏爱好者
- **使用场景**：
  1. 短时间休闲娱乐，打发碎片时间
  2. 挑战不同难度AI，提升策略思维
  3. 与其他玩家比较战绩，竞争排行榜
  4. 理解三维空间逻辑，锻炼空间想象力

## 用户操作流程设计

### 功能 F-001：3D棋盘渲染与视角控制

**前置条件**：
- 用户状态：已进入游戏界面
- 系统状态：棋盘已初始化
- 依赖功能：无

**操作步骤**：
| 步骤 | 用户操作 | 系统响应 | 异常处理 |
|------|----------|----------|----------|
| 1 | 查看棋盘 | 显示5x5x6三维棋盘，简约视觉风格（高度可扩展至8层） | - |
| 2 | 右键按住拖拽 | 棋盘视角随鼠标移动平滑旋转 | 若左键点击则触发棋子放置（非视角操作） |
| 3 | 松开右键 | 视角固定在当前位置 | - |

**操作结果**：
- 成功标志：用户可从任意角度观察棋盘
- 后续操作：可继续旋转或进行棋子放置
- 可逆性：视角可随时调整，无限制

---

### 功能 F-002：棋子放置

**前置条件**：
- 用户状态：当前为玩家回合
- 系统状态：游戏进行中，棋盘有可用空位
- 依赖功能：F-001 已显示棋盘

**操作步骤**：
| 步骤 | 用户操作 | 系统响应 | 异常处理 |
|------|----------|----------|----------|
| 1 | 左键点击棋盘某位置 | 检测点击位置是否有效 | 若位置已满则提示"此处无法放置" |
| 2 | 确认有效 | 棋子下落到该列最底层空位，播放简单下落动画 | - |
| 3 | 下落完成 | 切换到AI回合，显示"AI思考中"提示 | - |

**操作结果**：
- 成功标志：棋子正确落位，回合切换
- 后续操作：等待AI行动
- 可逆性：不可撤销（无悔棋功能）

---

### 功能 F-003：胜负判定

**前置条件**：
- 用户状态：游戏进行中
- 系统状态：刚完成一次棋子放置
- 依赖功能：F-002 棋子已放置

**操作步骤**：
| 步骤 | 用户操作 | 系统响应 | 异常处理 |
|------|----------|----------|----------|
| 1 | 无（自动触发） | 系统检测13个方向向量下的所有连线路径 | - |
| 2 | 若发现4子连线 | 触发胜负特效，显示胜利/失败结果 | 若棋盘满且无连线则显示"平局" |
| 3 | 无连线 | 继续游戏，等待下一步 | 若棋盘已满则判定平局 |

**操作结果**：
- 成功标志：正确判定胜负/平局
- 后续操作：触发F-005游戏结束流程
- 可逆性：不可逆

---

### 功能 F-004：AI对战

**前置条件**：
- 用户状态：当前为AI回合
- 系统状态：游戏进行中，显示"AI思考中"
- 依赖功能：F-002 玩家已完成操作

**操作步骤**：
| 步骤 | 用户操作 | 系统响应 | 异常处理 |
|------|----------|----------|----------|
| 1 | 无（自动触发） | AI根据难度计算最佳位置 | - |
| 2 | 等待AI思考 | 显示思考动画/提示，耗时根据难度变化 | - |
| 3 | AI决策完成 | AI放置棋子，触发F-003胜负判定 | - |

**操作结果**：
- 成功标志：AI棋子正确落位
- 后续操作：切换回玩家回合
- 可逆性：不可撤销

**难度设计**：
| 难度等级 | 思考深度 | 失误率 | 备注 |
|----------|----------|--------|------|
| 简单 | 1-2步 | ~30%随机选次优解 | 适合新手 |
| 中等 | 2-3步 | ~10%随机选次优解 | 有一定挑战 |
| 困难 | 4-5步 | ~0%始终最优 | 高难度挑战 |

### 功能 F-005：游戏流程控制

**前置条件**：
- 用户状态：已登录或开始游戏
- 系统状态：在主菜单或游戏结束状态
- 依赖功能：F-006（若需登录）

**操作步骤**：
| 步骤 | 用户操作 | 系统响应 | 异常处理 |
|------|----------|----------|----------|
| 1 | 进入游戏 | 显示难度选择（简单/中等/困难） | - |
| 2 | 选择难度 | 显示先后手选择（先手黑棋/后手白棋/随机） | - |
| 3 | 选择先后手 | 开始游戏，进入F-001棋盘界面 | - |
| 4 | 游戏结束 | 显示胜负结果+特效，弹出选项："再来一局"/"返回主菜单" | - |
| 5 | 选择"再来一局" | 可重新选择难度和先后手，开始新局 | - |
| 6 | 选择"返回主菜单" | 返回F-006主菜单界面 | - |

**操作结果**：
- 成功标志：完整游戏流程闭环
- 后续操作：可继续游戏或退出
- 可逆性：结束后无法回到游戏过程

### 功能 F-006：用户系统（⏸️ 暂缓，后续版本再做）

*注：本功能暂缓开发，以下为后续版本参考设计。*

**前置条件**：
- 用户状态：首次使用或开始游戏
- 系统状态：在主菜单或游戏结束状态
- 依赖功能：无

**操作步骤**：
| 步骤 | 用户操作 | 系统响应 | 异常处理 |
|------|----------|----------|----------|
| 1 | 进入游戏 | 系统检查本地是否已有保存的用户名 | - |
| 2 | 若无用户名 | 生成临时ID，直接进入游戏流程 | - |
| 3 | 游戏结束，首次上榜 | 弹出输入名字对话框 | - |
| 4 | 输入用户名 | 敏感词审查，若通过则保存到本地localStorage | 若包含敏感词则提示"名字不合规，请修改" |
| 5 | 确认用户名 | 用户名保存本地，提交战绩到云端排行榜 | - |
| 6 | 后续游戏结束 | 直接使用已保存的用户名提交战绩 | 若网络失败则提示"提交失败，稍后重试" |

**操作结果**：
- 成功标志：用户名本地持久化，战绩成功提交云端
- 后续操作：可查看排行榜
- 可逆性：可修改本地用户名（重新输入）

**用户名规则**：
- 长度：1-12字符
- 允许：中文、英文、数字、常用符号
- 禁止：敏感词、特殊字符
- 同一名字可被多人使用，不强制唯一

### 功能 F-007：战绩记录（本地）

**前置条件**：
- 用户状态：游戏进行中或刚结束
- 系统状态：胜负已判定
- 依赖功能：F-003胜负已判定

**操作步骤**：
| 步骤 | 用户操作 | 系统响应 | 异常处理 |
|------|----------|----------|----------|
| 1 | 无（自动触发） | 游戏结束后，系统更新本地战绩数据 | - |
| 2 | 查看战绩 | 在UI面板或战绩页面显示统计数据 | - |

**操作结果**：
- 成功标志：战绩数据正确更新并展示
- 后续操作：可继续游戏
- 可逆性：不可撤销

**记录数据字段**：
| 字段 | 说明 | 示例 |
|------|------|------|
| 简单难度胜场 | 简单难度下获胜次数 | 5 |
| 简单难度败场 | 简单难度下失败次数 | 2 |
| 简单难度胜率 | 简单难度胜场/(胜场+败场) | 71.4% |
| 中等难度胜场 | 中等难度下获胜次数 | 3 |
| 中等难度败场 | 中等难度下失败次数 | 4 |
| 中等难度胜率 | 中等难度胜场/(胜场+败场) | 42.9% |
| 困难难度胜场 | 困难难度下获胜次数 | 1 |
| 困难难度败场 | 困难难度下失败次数 | 5 |
| 困难难度胜率 | 困难难度胜场/(胜场+败场) | 16.7% |
| 总胜场 | 所有难度胜场之和 | 9 |
| 总败场 | 所有难度败场之和 | 11 |
| 总胜率 | 总胜场/(总胜场+总败场) | 45% |

**数据存储**：
- 使用 localStorage 持久化存储
- 每局结束后自动更新
