## 需求概述

在现有 3D 四子棋游戏中增加主题系统，支持三套主题（经典、猫咪、机甲）切换。棋子、棋盘、环境随主题变化呈现不同的视觉风格和动画效果。主题切换在主菜单进行，带有预览和确认机制。

## 主题定义

### 主题列表

| 主题 ID | 名称 | 描述 |
|---------|------|------|
| CLASSIC | 经典主题 | 苹果风格：纯净白棋 + 深空灰棋，毛玻璃底座，极简设计 |
| CAT | 猫咪主题 | 黑猫白猫对战，蹲坐/趴睡姿态 |
| MECHA | 机甲主题 | 机甲对战，站立/收拢姿态 |

### 经典主题设计细节

| 元素 | 设计风格 | 具体描述 |
|------|----------|----------|
| **棋子** | 苹果极简 | 纯净白色棋子 + 深空灰棋子，圆柱体，高光材质，边缘有柔和光晕 |
| **底座** | 毛玻璃质感 | 半透明磨砂底座，圆角矩形，白色/浅灰配色 |
| **网格线** | 极简线条 | 细线条，浅灰色，低对比度，不抢眼 |
| **背景** | 渐变干净 | 浅灰渐变（上浅下深），干净无装饰 |
| **光照** | 柔和均匀 | 柔和白光，无强烈阴影，营造通透感 |
| **动画** | 精致细腻 | 微妙的呼吸缩放、光晕脉动、纯净拖尾 |
| **高亮** | 光晕增强 | 悬停/胜利时边缘光晕变亮，不刺眼 |
| **特效** | 纯净优雅 | 光晕拖尾（非粒子爆炸），涟漪扩散 |

### 主题切换

- **入口**：主菜单独立按钮
- **预览方式**：文字描述 + 棋子默认图预览
- **切换确认**：二次确认弹窗
- **切换时机**：游戏结束后可切换（游戏进行中不可切换）

## 动画系统设计

### 状态机定义

```
┌─────────────────────────────────────────────────────────────┐
│ 棋子状态机                                                   │
├─────────────────────────────────────────────────────────────┤
│                                                              │
│  ┌──────────┐                                                │
│  │  休眠    │ ← 被子覆盖（非顶层）                             │
│  │  (Sleep) │   静态休眠姿态，无动画                            │
│  └────┬─────┘                                                │
│       │ 被移除覆盖                                            │
│       ↓                                                      │
│  ┌──────────┐                                                │
│  │  活跃    │ ← 顶层可被接触                                   │
│  │  (Idle)  │   呼吸节律循环动画（2秒/次）                      │
│  └────┬─────┘                                                │
│       │ 鼠标悬停该位置                                         │
│       ↓                                                      │
│  ┌──────────────┐                                            │
│  │  悬停响应     │ ← 区分己方/对方棋子                          │
│  │  (Hover)     │   每次悬停都播放，可重复播放                    │
│  └──────┬───────┘                                            │
│         │                                                    │
│  ┌──────┴───────┐  新棋子下落                                │
│  │              │                                            │
│  ↓              ↓                                            │
│  ┌──────────┐  ┌──────────┐                                 │
│  │ 下落动画  │  │ 对抗动画  │ ← 最顶层直面冲击的棋子             │
│  │ (Fall)   │  │ (Impact) │   区分己方/对方                    │
│  └──────────┘  └──────────┘                                 │
│       │              │                                       │
│       └──────┬───────┘                                       │
│              ↓                                               │
│       落地后恢复活跃状态                                        │
│                                                              │
│  ┌──────────┐  游戏结束                                      │
│  │ 胜负动画  │ ← 区分己方（胜利）/对方（失败）                   │
│  │ (Win/Lose)│   所有胜利/失败棋子播放                         │
│  └──────────┘                                                │
│                                                              │
└─────────────────────────────────────────────────────────────┘
```

### 状态 1：活跃（Idle）- 呼吸节律动画

顶层棋子持续播放的循环动画，2 秒/次循环。

| 主题 | 是否激活 | 表现形式 |
|------|----------|----------|
| 经典 | ✅ | 微弱呼吸缩放 + 边缘光晕脉动 |
| 猫咪 | ✅ | 蹲坐姿态，动尾巴 + 动耳朵（不一定要全都有） |
| 机甲 | ✅ | 站立姿态，机械部件开合 + 灯光流动 |

---

### 状态 2：休眠（Sleep）- 静态素材切换

被子覆盖的棋子，切换为休眠素材，颜色和透明度不变。

| 主题 | 休眠姿态 |
|------|----------|
| 经典 | 无变化（无休眠素材） |
| 猫咪 | 趴睡姿态（从蹲坐变为趴下） |
| 机甲 | 重叠收拢姿态（机械部件折叠收拢） |

---

### 状态 3：悬停响应（Hover）- 互动动画

鼠标悬停到某个位置时，该列**最顶层**棋子产生响应动画（区分己方/对方）。每次悬停都播放，可重复播放。

| 主题 | 己方棋子 | 对方棋子 |
|------|----------|----------|
| 经典 | 微缩放 + 发光增强（边缘光晕变亮） | 微缩放 + 颜色变暗（警示感） |
| 猫咪 | 看向鼠标位置 + 翻身露出肚皮 | 看向鼠标位置 + 伸爪拍打鼠标动作 |
| 机甲 | 看向鼠标位置 + 微蹲准备迎接（灯光流动加速） | 看向鼠标位置 + 左盾右剑半举威胁动作 |

**触发规则**：
- 仅该列正下方的棋子响应（不波及相邻列）
- 每次悬停都播放动画（可重复触发）
- 悬停期间持续，离开时恢复

---

### 状态 4：下落动画（Fall）- 棋子下落

新棋子下落时，下落棋子自身有姿态动画。

| 主题 | 下落动画 | 特效 |
|------|----------|------|
| 经典 | 重力下落 | 纯净光晕拖尾（白棋白色/黑棋浅蓝） |
| 猫咪 | 伸腿下落姿态 | 暂无特殊特效 |
| 机甲 | 俯冲姿态 | 暂无特殊特效 |

**机甲特殊规则**：
- 底部是**己方**棋子 → 举盾俯冲
- 底部是**敌方**棋子 → 举剑俯冲

---

### 状态 5：对抗动画（Impact）- 承接棋子反应

棋子落地时，**最顶层**（直面冲击的）承接棋子产生对抗动画（区分己方/对方）。动画时长与下落时间相同。

| 主题 | 己方棋子（承接着落棋子） | 对方棋子（承接着落棋子） |
|------|--------------------------|--------------------------|
| 经典 | 微下沉 + 光晕扩散（涟漪效果） | 微下沉 + 颜色变暗（警示感） |
| 猫咪 | 原地侧弯躲避动作 | 炸毛动作 |
| 机甲 | 巨盾迎接冲击姿态 | 举剑威胁动作 |

**机甲特殊规则**：
- 己方 → 举盾姿态
- 敌方 → 举剑姿态

---

### 状态 6：胜负动画（Win/Lose）- 游戏结束

所有胜利/失败棋子播放对应动画，时长 3-5 秒。

| 主题 | 胜利动画（己方） | 失败动画（对方） |
|------|------------------|------------------|
| 经典 | 发光增强 + 轻微跳跃 | 颜色淡化（透明度降低） |
| 猫咪 | 左右跳跃 | 钻洞露出一个头 |
| 机甲 | 双臂高举欢呼状 | 能量熄灭破损 |

## 用户操作流程设计

### 功能 F-T01：主题切换

**前置条件**：
- 用户状态：在主菜单界面
- 系统状态：游戏未开始或已结束

**操作步骤**：

| 步骤 | 用户操作 | 系统响应 | 异常处理 |
|------|----------|----------|----------|
| 1 | 点击"主题切换"按钮 | 弹出主题选择面板 | - |
| 2 | 查看主题列表 | 显示三套主题：文字描述 + 棋子默认图 | - |
| 3 | 选择目标主题 | 高亮选中主题 | - |
| 4 | 点击"确认切换" | 弹出二次确认弹窗："确定切换到 [主题名] 吗？" | - |
| 5 | 确认 | 切换主题，关闭面板，更新界面预览 | 若素材加载失败则提示"主题加载失败，使用默认主题" |
| 6 | 取消 | 关闭弹窗，保持当前主题 | - |

**操作结果**：
- 成功标志：主题切换完成，新主题生效
- 后续操作：可开始新游戏
- 可逆性：随时可切换回其他主题

---

### 功能 F-T02：游戏内动画播放

**前置条件**：
- 用户状态：游戏进行中
- 系统状态：已选择主题，棋子已放置

**动画触发流程**：

| 触发时机 | 棋子状态 | 动画类型 | 说明 |
|----------|----------|----------|------|
| 棋子成为顶层 | 休眠→活跃 | 激活呼吸节律 | 三套主题都播放（经典：缩放+光晕，猫咪/机甲：各自动画） |
| 棋子被覆盖 | 活跃→休眠 | 切换休眠素材 | 猫咪变趴睡，机甲收拢 |
| 鼠标悬停 | 活跃→悬停响应 | 己方/对方不同动画 | 每次都播放，可重复 |
| 新棋子下落 | -→下落 | 下落动画 | 每套主题姿态不同 |
| 棋子落地 | 活跃→对抗 | 承接棋子对抗动画 | 己方/对方不同反应 |
| 游戏结束 | -→胜负 | 胜利/失败动画 | 所有棋子播放对应动画 |

## 非功能性需求

### 性能要求
- 待机动画帧率≥30fps
- 动画切换无卡顿
- 主题切换加载时间≤3 秒
- 素材内存占用可控（单主题≤10MB）

### 兼容性要求
- 模型格式优先 GLB/GLTF
- 找不到素材时 AI 生成
- 加载失败自动 fallback 到经典主题

### 可用性要求
- 主题预览直观（文字 + 图片）
- 切换确认清晰（二次弹窗）
- 加载失败提示友好

## 边界与约束

**功能边界**：
- ✅ 包含功能：
  - 三套主题（经典-苹果风格、猫咪、机甲）
  - 棋子模型/纹理/材质切换（经典主题含毛玻璃质感、高光材质）
  - 完整动画系统（呼吸、悬停、下落、对抗、胜负）
  - 棋盘主题渲染（经典主题含毛玻璃底座、极简网格线）
  - 环境主题渲染（经典主题含浅灰渐变背景、柔和光照）
  - 主题切换 UI
  - 素材加载管理

- ❌ 不包含功能：
  - 自定义主题（用户自己上传模型）
  - 主题商城/购买
  - 音效（仅视觉动画）
  - 游戏进行中切换主题

**约束条件**：
- 素材来源：免费素材优先，找不到时 AI 生成
- 动画实现：代码动画优先，模型自带动画可选
- 性能：60fps 优先，简化效果保流畅
