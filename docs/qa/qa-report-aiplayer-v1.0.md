## 问题清单

### 问题1: E-4 边界阻挡的2连扩展 - AI优先双2连而非单线扩展

| 项目 | 内容 |
|------|------|
| **严重程度** | 中 |
| **涉及难度** | EASY, MEDIUM, HARD |
| **现象** | 棋盘只有两条2连WHITE，AI选择(1,0)创建双2连(POTENTIAL_DOUBLE=300分)，而非测试期望的(2,0)单线扩展 |
| **原因分析** | Layer 2 的 POTENTIAL_DOUBLE_OWN(300) 评分高于单条2连扩展，AI选择了能创建双2连的位置而非单线扩展端 |
| **影响** | EASY 下此场景AI选择了"更高价值"位置，从游戏角度看并非错误选择，但测试断言过于严格 |
| **建议** | 评估函数行为合理（双2连确实比单2连有价值），测试断言需调整：允许AI选择任何"合理"的扩展位置 |

### 问题2: E-5 对方阻挡的2连扩展 - AI选择同列而非预期扩展端

| 项目 | 内容 |
|------|------|
| **严重程度** | 中 |
| **涉及难度** | EASY, MEDIUM, HARD |
| **现象** | 白棋(0,2),(0,3)，上端(0,4)被黑阻挡，AI选择(0,2)而非期望的(0,0)/(0,1) |
| **原因分析** | 实际是选择了(0,2)列中重力允许的最低位置，但validate断言 y>=0 && y<=1 |
| **影响** | AI选择了(0,2)位置（已有白棋的列），测试预期为(0,0)或(0,1) |
| **建议** | 检查validate逻辑和重力合规性 |

### 问题3: M-5 双线叉子创建 - AI未选择叉子交汇点

| 项目 | 内容 |
|------|------|
| **严重程度** | 高 |
| **涉及难度** | MEDIUM, HARD |
| **现象** | 白棋(0,2),(1,2),(2,1)已布好，下(2,2)应形成双3连获胜，但AI未选择此位置 |
| **原因分析** | 当前评估Layer3降权(×0.05)可能影响叉子识别；Minimax depth=3/4可能发现此位置，但静态评估未给足够分 |
| **影响** | MEDIUM和HARD均失败，HARD下仍失败说明Minimax也未发现 |
| **建议** | 检查叉子检测逻辑是否遗漏此场景；验证Layer2双威胁检测是否能识别 |

### 问题4: M-7/H-7 3D方向+重力合规 - AI未选择对角线扩展端

| 项目 | 内容 |
|------|------|
| **严重程度** | 中 |
| **涉及难度** | MEDIUM, HARD |
| **现象** | 白棋(1,1,z=0),(2,2,z=1)已布好，(3,3)列z=0,z=1有黑棋，AI应选(3,3)扩展对角线 |
| **原因分析** | 对角线1,1,1的LineIndex可能未正确记录此2连（z坐标跨越问题）；AI可能选择了其他更有价值的位置 |
| **影响** | 3D方向扩展测试失败 |
| **建议** | 检查LineIndex对角线记录和Layer3位置统计是否正确处理3D场景 |

### 问题5: H-5 3方向3D叉子 - AI未在交汇点落子

| 项目 | 内容 |
|------|------|
| **严重程度** | 高 |
| **涉及难度** | HARD |
| **现象** | 白棋水平(1,2)+垂直(2,2,z=0,1)+对角(1,1,z=0)，下(2,2)应形成多线获胜，AI未选择 |
| **原因分析** | 与M-5类似，叉子交汇点评分不足；垂直2连在z=0,1可能未被LayerIndex正确识别为"可下"位置 |
| **影响** | HARD最难场景失败 |
| **建议** | 检查LayerIndex对垂直方向2连的处理（z=0,1是否有openEnds） |

### 问题6: H-8 反叉子 - AI未选择(2,2)阻挡点

| 项目 | 内容 |
|------|------|
| **严重程度** | 中 |
| **涉及难度** | HARD |
| **现象** | 黑棋两条2连交汇(2,2)，AI应选(2,2)同时阻挡和创造己方叉子 |
| **原因分析** | 阻挡评分(BLOCK=300) vs 己方2连扩展评分，AI选择了其他位置 |
| **影响** | 反叉子场景AI未同时考虑攻防 |
| **建议** | 检查Layer3防守叉子倍率(×2)是否生效；可能需增加阻挡叉子的额外加分 |

## 测试结论

**结论：有条件通过**

### 通过项
- ✅ 32/42 用例通过（76.2%）
- ✅ 基础功能：立即获胜、阻挡、3D垂直方向、双3连威胁、防叉点、性能均正常
- ✅ 难度配置正确（EASY=2, MEDIUM=3, HARD=4）
- ✅ 失误率机制正常（HARD 100%最优）
- ✅ 响应时间达标（HARD < 4s）

### 未通过项（6个场景，10次失败）
- ⚠️ 均为棋谱场景测试失败，集中在2连扩展、3D叉子识别、反叉子场景
- ⚠️ 问题根因分为两类：
  - **测试断言过严**：E-4 场景AI选择双2连创建是合理决策，测试应放宽
  - **评估函数缺陷**：M-5/H-5 叉子交汇点评分不足，M-7 3D对角线未正确识别，H-8 防守叉子倍率可能未生效

### 建议
1. **P0（高）**：修复 M-5（双线叉子）、H-5（3D叉子）评估逻辑，确保叉子交汇点被正确评分
2. **P1（中）**：调整 E-4/E-5 测试断言，允许AI选择多个合理扩展位置
3. **P1（中）**：检查 M-7 3D对角线的LineIndex记录是否正确
4. **P2（低）**：H-8 反叉子场景优化防守叉子评分
