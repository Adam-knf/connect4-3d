## 一、棋盘扩展性约束体系

在3D四子棋中，"威胁线"的扩展性受三个因素制约，现有测试缺乏对这些约束的覆盖。

### 约束因子总览

| 约束因子 | 影响 | 判定依据 |
|----------|------|----------|
| **A. 棋盘边界** | 4连段超出棋盘范围的一端不可扩展 | 扩展坐标 `(x,y,z)` 是否在 `[0,4]×[0,4]×[0,5]` 内 |
| **B. 对方棋子阻挡** | 4连段扩展端被对手棋子占据 | `board.getPiece(extPos) !== 'EMPTY'` |
| **C. 重力合规性** | 扩展端z>0时下方必须已有棋子铺垫 | `board.findDropPosition(x,y) === targetZ` |

### 开放端计算规则

```
openEnds = 0         ← 两端都不可扩展
openEnds = 1         ← 一端可扩展
openEnds = 2         ← 两端都可扩展

readyEnds ⊆ openEnds ← 其中满足重力可直接下的端数量
```

### 各评估层的响应

| 评估层 | 对 openEnds 的依赖 | 当前处理 |
|--------|-------------------|----------|
| Layer 0 (立即胜负) | **不检查** — quickWouldWinFast 直接验证放置后是否连4 | 无条件返回 WIN |
| Layer 1 (3连) | 要求 `line.openEnds > 0` — 否则不给THREE_OWN分 | ✅ 正确 |
| Layer 2 (2连) | 要求 `line.openEnds > 0` — 否则不计入aiTwoLines | ✅ 正确 |
| evaluateInternalNode | 不检查 `openEnds` — 改用 `findExtensionCells` | ✅ Route C已修复 |
| Layer 3 (PotentialFork) | 检查 `calculateLineDiscount` — 不可下端越多折扣越大 | ✅ 有折扣处理 |

### 缺失的测试维度

```
边界阻挡:  2连/3连因棋盘边界导致 openEnds 减少 → 无测试覆盖
对方阻挡:  2连/3连因对方棋子导致 openEnds=0 → 无测试覆盖
重力铺垫:  3D方向扩展需重力合规 → 无测试覆盖
```

## 三、EASY 级别新增（4个）

**能力**: Layer 0-2, depth=2, 无Minimax

### E-3: 垂直3连获胜（3D独有方向）

验证垂直方向 `(0,0,1)` 的 3连 → 立即获胜。重力合规：目标列z=0已空，AI可下至z=0。但AI的实际目标z=3，需要z=0,1,2被填充。

**棋局**（使用 `board.setPiece` 预填充）:
```
board.setPiece({x:2,y:2,z:0}, 'WHITE');
board.setPiece({x:2,y:2,z:1}, 'WHITE');
board.setPiece({x:2,y:2,z:2}, 'WHITE');
// (2,2)列z=3空 — AI placePiece(2,2) → z=3 → 垂直4连
Black(0,0) // 填充
```

**预期**: AI 选择 (2,2) → `winResult.winner === 'WHITE'`
**约束验证**: 垂直方向的openEnds计算正确，gravity 使第4子落到 z=3

---

### E-4: 边界阻挡的扩展方向选择

验证 AI 识别 "一端被边界阻挡的2连"，正确选择**唯一**可扩展端。

**棋局**:
```
X方向 segment: (0,0),(1,0),(2,0),(3,0)
AI 2连 at (1,0), (2,0)
左扩展端: (-1,0) — ❌ 边界阻挡(z=0层x=-1越界)
右扩展端: (4,0)  — ✅ 可下
```

```
White(1,0), White(2,0)   // 靠近x=0边缘的X方向2连
Black(4,0)               // 对手填充
```

**预期**: AI 选择 (0,0) 扩展（唯一可扩展方向）
**断言**: `result.x === 0 && result.y === 0`

---

### E-5: 对方棋子阻挡的扩展端

验证 AI 识别 "一端被对手棋子阻挡的2连"，选择未阻挡端。

**棋局**:
```
X方向 segment: (1,0),(2,0),(3,0),(4,0)
AI 2连 at (2,0), (3,0)
左扩展端: (1,0) — 但 (1,0) 已由AI占据（右移segment）
左扩展端: (0,0) ❌ 被Black占据
右扩展端: (5,0) — ❌ 边界阻挡
```

等等，让我重新设计。这个 segment 布局不太对。让我用更清晰的 setup：

```
Y方向 segment: (0,0),(0,1),(0,2),(0,3) 在 x=0, z=0
openEnds: 下端 (0,-1) — 边界阻挡
          上端 (0,4)  — 可下，除非被阻

AI 2连 at (0,2), (0,3)
     ↓ 放置后判定 "放置后的状态"
openEnds: (0,4) — 可下
          (0,-1) — 边界阻挡
所以 openEnds = 1 ✓

但如果 (0,4) 被 Black 占据：
openEnds = 0 → AI 不应选择这个扩展
```

**正确棋局**:
```
// 场景A: openEnds=1（一端被对手占，一端边界）
board.placePiece(0, 2, 'WHITE');
board.placePiece(0, 3, 'WHITE');
board.setPiece({x:0,y:4,z:0}, 'BLACK');  // 阻挡上端
// 此时 Y方向 segment(0,0)-(0,3) 中 AI有(0,2)(0,3)
// 但这是放置后的位置。我们需要看放置前AI评估什么...

// 修正：AI正在评估是否在(0,1)下子
// 放置前: AI pieces at (0,2),(0,3), Black at (0,4)
// Segment (0,0),(0,1),(0,2),(0,3)
// aiCountAfter(放置(0,1)): (0,1)下后 aiCount=3
```

Hmm, 这有点绕。让我换个更清晰的思路：

E-5 测试的是：AI 评估候选位置时，该位置形成 2连的 line 有 openEnds=0（扩展端被对手占据），但另一个候选位置形成的 2连有 openEnds=1。AI 应当选择 openEnds=1 的位置。

```
// 候选A: (0,0) — 形成 X方向 2连但 openEnds=0
// Segment: (0,0),(1,0),(2,0),(3,0)
// AI pieces at (0,0),(1,0) — 放(0,0)前已有(1,0)...不对
```

我太纠结于精确的数学验证了。让我用更简明的设计方法：

### E-5 (简化): 对方阻挡2连 vs 己方可发展2连

**概念**: 棋盘上有两条2连：
- 线路A: AI 2连，但扩展端被对手占据 → openEnds=0 → 不计入aiTwoLines
- 线路B: AI 2连，扩展端空 → openEnds>=1 → 计入aiTwoLines

AI 应选择线路B的扩展，而非线路A。

**具体 setup**（由 Dev 实现时验证坐标）:
```
线路A: AI 有2子在4连段内，扩展端唯一可下位置被对手占据
线路B: AI 有2子在4连段内，扩展端可下
```

### E-7: 两端都阻挡的2连无价值

验证 AI 不应选择"两端都不可扩展"的2连。

**棋局**:
```
Y方向 segment(0,0)-(0,3):
Black at (0,0), AI at (0,1),(0,2), Black at (0,3)

AI 2连 at (0,1),(0,2)。两端都被阻挡。
openEnds = 0 → 不计入 Layer 2 威胁线
```

对 AI 来说，扩展 2连 毫无价值（TWO_OWN 不触发）。AI 应选择其他位置发展。
同样由 Dev 实现时细化坐标。

## 四、MEDIUM 级别新增（4个）

**能力**: Layer 0-3 + Minimax depth=3

### M-5: 双线叉子创建

创建标准叉子（2条3连威胁交汇于空位）。

**棋局概念**: AI 有两条方向不同的 2连，交汇于空位 P。落子 P 即形成双3连。
**断言**: 落子后 AI 获胜（对手只能堵一路）

---

### M-6: 提前阻挡对手叉点

对手有两条 2连交汇于空位 P。AI 不占据 P 则对手下一步形成双3连必胜。

**棋局概念**:
- 对手线A: 2连，延伸需经过 P
- 对手线B: 2连，延伸需经过 P
- AI 需抢占 P
**断言**: `result.x === P.x && result.y === P.y`

---

### M-9: 3连因边界阻挡不可获胜

Layer 1 检查 `openEnds > 0`。若3连的唯一扩展端在边界外，不应计为有效威胁。

**棋局概念**:
```
AI 3连在靠近边界的线上
唯一可扩展方向超出棋盘范围 → openEnds=0
AI 不应认为该线有威胁价值
```

与 E-4 的边界阻挡概念相同，但作用于 Layer 1 的3连而非 Layer 2 的2连。

**测试**: 对比"有 openEnds 的 3连" vs "openEnds=0 的 3连"的评分。

---

### M-7: 3D方向 + 重力合规的威胁扩展

测试 3D 方向 (1,0,1) 或 (1,1,1) 的 openEnds 判定，以及 gravity 对 readyEnds 的影响。

**棋局概念**:
```
在 3D 方向上有 2连，扩展端在 z>0
扩展端下方已填充 → 可直接下 (ready)
同一方向的另一端在 z=0 → 也可直接下
```

需要 `board.setPiece` 预填充使目标 z 层 gravity-compliant。

## 五、HARD 级别新增（3个）

**能力**: depth=4 Minimax + Layer 0-3 + forkScoreMultiplier=1.5

### H-5: 3方向3D叉子（含垂直+重力）

3D 空间中的复杂叉子：AI 在 3 条不同方向上（含垂直方向）各有威胁，交汇于同一个 gravity-compliant 的空位。

**方向组合示例**: 水平 `(1,0,0)` + 垂直 `(0,0,1)` + 对角 `(1,1,0)`
**断言**: AI 选择交汇点，验证 3D 方向 + gravity 的综合评估正确

---

### H-6: 3步必胜序列（depth=4 搜索深度优势）

AI 需要 depth=4 搜索才能发现的必胜路线。

**设计约束**:
- 不存在即时获胜（避免 Layer 0 截断）
- 不存在显性叉子（避免 InternalNode 加分过多）
- 对手每步只有唯一最优防守
- 路线: AI 走 A → 对手被迫走 B → AI 走 C → 对手被迫走 D → AI 在 E 获胜

---

### H-8: 反叉子（阻挡对手的同时创造己方叉子）

AI 阻挡对手叉点，同时该位置也是 AI 自身的叉子起点。

**棋局概念**:
```
对手线A: 2连 → 扩展需经过 P
对手线B: 2连 → 扩展需经过 P
AI 线A: 1子 → 加上 P 形成 2连且扩展端空
AI 线B: 1子 → 加上 P 形成 2连且扩展端空
P: 空位，AI 下 P 即同时阻挡对手 + 创造自己叉子
```

## 八、Dev 实现指引

### Gravity 填充模板

当需要 AI 下到 z>0 时，使用以下模板预填充：

```typescript
// 填充列 (targetX, targetY) 的 z=0..targetZ-1
for (let zz = 0; zz < targetZ; zz++) {
  board.setPiece({ x: targetX, y: targetY, z: zz }, 'BLACK');
}
// 验证
const dropZ = board.findDropPosition(targetX, targetY);
console.assert(dropZ === targetZ, `Expected z=${targetZ}, got ${dropZ}`);
```

### openEnds 诊断

每个棋局 setup 后，对关键 line 做 openEnds 诊断：

```typescript
const lineIds = board.getLineIdsAtPosition({x, y, z});
for (const lid of lineIds) {
  const line = board.getLineRecord(lid);
  if (line) console.log(`Line ${lid}: openEnds=${line.openEnds}, readyEnds=${line.readyEnds}, dir=(${line.direction.x},${line.direction.y},${line.direction.z})`);
}
```
